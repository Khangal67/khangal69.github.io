Unit 1: Data

  • Bit: A binary digit, 0 or 1.
  • Byte: A group of 8 bits.
  • Roundoff: Errors from not enough bits to store precision.
  • Analog Data: Smooth, continuous values like sound.
  • Lossless: Compression that keeps all original data.
  • Lossy: Compression that removes some data permanently.
  • Metadata: Data about data, like file info.

Unit 2: Computer Systems and Networks

  • Computing Device: Any device that runs a program.
  • Computer Network: Devices connected to send/receive data.
  • Path: The series of connections data travels.
  • Protocol: Rules for how systems communicate.
  • Fault Tolerant: Keeps working even when parts fail.
  • Scalability: Ability to grow or shrink with demand.
  • Bandwidth: Max data transfer rate.
  • Packets: Small chunks of data messages.
  • Routers: Forward data across networks.
  • DNS: Translates names like google.com into IP addresses.

Domain Name System (DNS)

DNS is like the internet’s address book. When you type a web address (like www.example.com) into your browser, DNS translates it into the computer-friendly IP address so your device can connect to the server.

DNS works in several steps:

  1. Your computer asks a DNS resolver to find the IP.
  2. If unknown, the resolver asks a root server, which points to the top-level domain (TLD) server like .com.
  3. The TLD server directs the request to the authoritative name server with the final IP.
  4. The resolver returns the IP to your computer, which then connects to the website.

Fun fact: DNS was created in 1983 to replace a single text file listing every computer on the early internet. Today, it uses a worldwide network of servers to keep sites reachable and fast.

Connection to Units 1 & 2: DNS uses data in the form of packets, relies on protocols like TCP/UDP, and depends on routers and fault tolerance in networks to function reliably.
